Принцип единственной ответственности (SRP)
Формулировка: Класс должен иметь только одну причину для изменения. Это значит, что объект должен отвечать только за одну конкретную функциональность внутри системы.

1. Пример нарушения принципа
Представим систему интернет-магазина. У нас есть класс Order (Заказ), который выполняет слишком много задач: хранит данные, считает стоимость и записывает информацию в базу данных.

Python
class Order:
    def __init__(self, items):
        self.items = items

    def calculate_total(self):
        # Логика расчета стоимости
        return sum(item.price for item in self.items)

    def save_to_database(self):
        # Логика работы с БД (нарушение SRP!)
        print(f"Сохраняем заказ в базу данных...")
Почему здесь нарушен SRP?
Этот класс имеет две причины для изменения:

Если изменится логика расчета цены (бизнес-логика).

Если изменится способ хранения данных (инфраструктура, например, переход с SQL на MongoDB).

2. Исправление (Рефакторинг)
Мы разделим ответственность между двумя классами: Order будет отвечать только за бизнес-логику заказа, а OrderRepository — только за операции сохранения.

Python
# Класс отвечает только за данные заказа
class Order:
    def __init__(self, items):
        self.items = items

    def calculate_total(self):
        return sum(item.price for item in self.items)

# Отдельный класс отвечает только за сохранение
class OrderRepository:
    def save(self, order: Order):
        print(f"Сохраняем заказ в базу данных...")
3. Комментарий к исправлению
Разделение логики: Теперь класс Order «ничего не знает» о базе данных. Он стал чище и легче для тестирования.

Гибкость: Если нам потребуется изменить способ сохранения (например, вместо базы данных отправлять заказ в файл или облачное хранилище), нам не нужно будет затрагивать класс Order.

Соблюдение SRP: Мы привели код к состоянию, где каждый класс выполняет только одну задачу. Это значительно упрощает поддержку кода, так как изменения в одной части системы (например, БД) не вызывают побочных эффектов в другой (расчет стоимости).

Принцип единственной ответственности (SRP) является первым из пяти принципов SOLID и служит фундаментом для создания модульного и поддерживаемого кода.
