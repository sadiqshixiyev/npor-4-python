## Описание задания
Спроектировать и реализовать архитектурный шаблон «Декоратор» для трансформации потока данных. Базовый компонент выполняет получение актуальных курсов валют с удаленного API Центрального Банка Российской Федерации в формате JSON. Группа конкретных декораторов расширяет функционал, позволяя приводить данные к валидным форматам JSON (с форматированием отступов), YAML и плоским таблицам CSV, а также осуществлять экспорт в локальные файлы конфигураций.

---

## Архитектура объектной модели

Программа спроектирована с жестким соблюдением абстракций и интерфейсов объектно-ориентированного программирования:
* **`Component` (ABC)**: Абстрактный базовый класс, объявляющий обязательный интерфейс работы с текстовыми структурами через метод `operation()`.
* **`CurrencyComponent`**: Реализация бизнес-логики. Осуществляет HTTP-запросы (с помощью библиотеки `requests`) к эндпоинту ЦБ РФ.
* **`BaseDecorator`**: Базовый абстрактный слой надстройки декораторов, хранящий ссылку на внутренний экземпляр `Component`.
* **`JsonDecorator`, `YamlDecorator`, `CsvDecorator`**: Конечные декораторы, выполняющие трансформацию данных и реализующие операции сохранения на диск (`save_to_file`).

## Оценка сложности алгоритмов
* **Временная сложность (Time Complexity)**: 
  * Получение данных: $O(1)$ в рамках сетевого запроса.
  * Форматирование JSON/YAML: $O(M)$, где $M$ — объем сериализуемой информации.
  * Преобразование в CSV: $O(K)$, где $K$ — количество парсерных объектов валют (линейный проход по элементам словаря).
* **Пространственная сложность (Space Complexity)**: $O(M)$ для размещения загруженных структур данных во временном буфере оперативной памяти текстовых процессоров.
