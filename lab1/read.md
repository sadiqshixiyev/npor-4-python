## Теоретические и практические особенности реализации

### 1. Итераторы (Задание 1)
* **Способ 1 (Класс-итератор):** Базируется на явной реализации методов `__iter__()` (возвращает сам объект) и `__next__()` (вычисляет следующее состояние). Преимущество — полный контроль над состоянием объекта. Недостаток — избыточность кода и необходимость вручную контролировать выброс исключения `StopIteration`.
* **Способ 2 (Функция-генератор):** Использует синтаксический сахар в виде ключевого слова `yield`. Python автоматически превращает функцию в объект-генератор, скрывая логику вызова `StopIteration` и сохранения состояния переменных между вызовами. Код становится значительно более читаемым.

### 2. Сопрограмма / Корутина (Задание 2)
Обычные генераторы могут только отдавать данные во внешний код. Сопрограммы используют выражение `n = yield response` для обеспечения **двусторонней связи**. Метод `.send(value)` передает значение внутрь выполняемого контекста функции и одновременно возобновляет ее работу до следующей инструкции `yield`.
Для удобства работы и исключения предварительного холостого вызова `next(gen)` написан декоратор `@prime_coroutine`, который автоматически подготавливает (стартует) корутину к приёму данных сразу в момент вызова.

## Сложность разработанных алгоритмов
* **Временная сложность (Time Complexity):** $O(n)$ для всех трех подходов. Каждый последующий элемент ряда Фибоначчи рассчитывается за фиксированное время $O(1)$ путем сложения двух предыдущих состояний. В корутине внутренний список `fib_list` доращивается только при необходимости, минимизируя лишние циклы вычислений при повторных или уменьшенных запросах.
* **Пространственная сложность (Space Complexity):** $O(1)$ для класса-итератора и функции-генератора (так как они выдают элементы потоком по одному), и $O(n)$ для корутины, которая вынуждена аккумулировать и хранить историю вычисленных чисел Фибоначчи внутри списка `fib_list` для мгновенной отдачи срезов.
